# 财务文件智能分析产品 TODO List

目标：从当前阶段推进到可交付给 100 人以内四大投资建议/交易咨询人员使用的个人 AI 助理 MVP。当前默认方案为飞书机器人上传优先，第一个能力是材料财务摘要，核心层需要保留后续扩展更多 AI 助理能力的口子。

## 0. 当前决策对齐

- [x] 确认 MVP 第一入口为飞书机器人消息。
- [x] 确认首批目标用户为 100 人以内的四大投资建议/交易咨询人员，主要诉求是客观反馈材料数据，辅助用户判断是否投资。
- [x] 确认产品定位：为受企业网络安全管控的专业人员提供外部网络个人 AI 助理服务，材料分析只是第一个能力。
- [x] 确认首批支持文件类型：Excel、CSV、可解析 PDF。
- [x] 确认图片、扫描件、图片型 PDF 暂不作为首批必做能力，OCR 后续按需接入。
- [x] 确认单文件大小上限：MVP 先限制为 20MB。
- [x] 确认首批分析场景：用户上传财务资源文件后生成财务摘要，不优先做银行流水、发票、合同或科目余额表专项分析。
- [x] 确认大模型供应商：MVP 暂定通义千问，但必须通过通用 LLM Provider 封装实现，后续只改 `baseURL`、`apiKey/appId`、`model` 等配置即可切换供应商。
- [x] 确认允许将文件内容发送给第三方大模型 API，但必须在隐私政策和用户提示中明确说明，并确认供应商默认不用于模型训练。
- [x] 确认数据留存策略：默认原始文件 30 天、解析文本 30 天、分析报告 180 天；实际策略通过数据库配置控制，由服务内定时任务自动清理。

## 2. 技术方案确认

## 6.1 上下文与记忆管理

### 6.1.0 已落地：MVP 多轮追问（手写实现，未用 LangChain/LangGraph）

- [x] 实现 `ConversationStore`（本地 JSON，以 open_id 为键）：每用户维护最近 N 个文件画像（文件名/摘要/关键字/分页路径）+ 每文件独立的多轮对话历史，超出按最近活跃时间 LRU 淘汰。
- [x] 实现文件画像生成：文件分析成功后用本地词频提取关键字 + 取报告首行摘要，写入会话存储（零额外模型调用）。
- [x] 实现两级检索：①问题与文件画像关键字重合度选文件（全不命中回退最近活跃文件）②文件内按页 TF 词频检索 Top-K 页（无命中回退前 K 页），保留页码引用。
- [x] 实现追问问答 `answer_question`：检索片段 + 最近 N 轮历史组装 messages 调用 `LLMProvider`，回复带页码引用。
- [x] 实现飞书消息路由：文件消息走原分析逻辑，文本消息走追问分支（`handle_message_receive`）。
- [x] 实现上下文窗口防膨胀：检索片段按 `qa_context_max_chars`(默认 24000) 字符截断 + 历史按 `qa_history_max_turns`(默认 5) 轮双重截断（写入与读取各截断一次）。
- [ ] 后续升级：检索从关键字/TF 升级为向量 embedding 或聚类，解决同义词/语义召回问题。
- [ ] 后续升级：超过窗口的旧对话用 LLM 摘要压缩，支持更长对话记忆。
- [ ] 后续升级：会话 JSON 并发写入加锁或迁移到 MySQL/Redis（当前与 TaskStore 同样无锁）。

### 6.1.1 规划（后续阶段）

- [ ] 短期策略：MVP 先只保留当前任务内上下文，不做长期记忆写入；先把文件级摘要、分片摘要和最终报告持久化，为后续追问做准备。
- [ ] 设计会话上下文模型 `ConversationContext`：记录用户当前任务、最近文件、最近报告、追问历史和可引用的中间结果。
- [ ] 设计长期记忆模型 `UserMemory`：只保存用户明确确认的偏好、常用分析口径、行业关注点和固定输出格式，不默认保存敏感材料内容。
- [ ] 设计文件级记忆模型 `DocumentMemory`：保存文件摘要、关键财务指标、页码引用、结构化数据表和可追问索引。
- [ ] 设计记忆写入策略：文件解析完成后先生成候选记忆，返回给用户确认“保留/舍弃/仅本次会话使用”。

- [ ] 设计追问上下文组装策略：优先放入当前文件摘要、相关页码证据、用户明确保留的偏好，再按 token budget 选择历史记忆。

## 13. MVP 完成标准

- [ ] 用户可以向飞书机器人发送文件。
- [ ] 系统可以自动创建分析任务。
- [ ] 系统可以解析 CSV、Excel、可解析 PDF。
- [ ] 系统可以调用大模型生成结构化报告。
- [ ] 用户可以收到飞书完成通知。
- [ ] 用户可以在飞书中收到 Markdown 格式报告。
- [ ] 报告包含文件概览、一句话摘要、财务要点摘要、全文概要、客观数据表和免责声明。
- [ ] 任务失败时有明确错误原因。
- [ ] 管理者可以查看日志并定位失败原因。
- [ ] 数据有备份，原始文件不公开暴露。
- [ ] 3 到 5 个种子用户能完成真实文件分析闭环。
